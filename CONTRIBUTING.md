# Contributing to Flutter Custom Updater

Thank you for your interest in contributing! 🎉 Your contributions help make this package better for everyone.

## 🚀 Ways to Contribute

1. **🐛 Report Bugs:** [Open an issue](https://github.com/VatanakChamroeun/flutter_custom_updater/issues) with clear reproduction steps.  
   Include screenshots, logs, or videos if possible.
2. **💡 Suggest Features:** Have an idea? Create an issue describing your proposed feature.
3. **📝 Improve Documentation:** Fix typos, clarify examples, or expand sections. Pull requests are welcome!
4. **🤝 Contribute Code:** Fix bugs, add new features, or improve performance. You can start by picking any open issue labeled `good first issue` or `help wanted`.

## 🧭 Getting Started

**Steps:**

1. Fork and clone the repo.
2. Create a branch:
   ```sh
   git checkout -b your-feature/your-bug-fix
   ```
3. Run the example app following the setup guide in the [README](./README.md#%EF%B8%8F-setup--run-).
4. Make your changes.

## 📬 Pull Requests

1. Push code to your-feature/your-bug-fix branch.
2. Open a Pull Request targeting the `develop` branch
3. In the PR description, clearly explain:
   - What was changed
   - Why it was changed
   - How to test or verify it

Please follow [Conventional Commits](https://www.conventionalcommits.org/) for commit messages (e.g., feat: add update dialog animation).

## 💬 Support

If you have any questions or issues, please [open an issue](https://github.com/vatanakchamroeun/flutter_custom_updater/issues) on GitHub.
